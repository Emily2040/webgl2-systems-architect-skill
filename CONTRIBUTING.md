# Contributing

Thanks for improving this skill. Keep changes focused on making WebGL2 agent work more accurate, measurable, and repeatable.

## Development checklist

1. Keep `SKILL.md` small and router-like.
2. Put detailed guidance in `references/` or `skills/core/`.
3. Update examples when routing, schema, or output behavior changes.
4. Update `CHANGELOG.md` and version references when behavior changes.
5. Run `python scripts/validate_repo.py` before opening a pull request.

## Pull request expectations

- Explain the user-visible behavior change.
- Name the module, schema, example, or fixture affected.
- Include validation output.
- Keep unrelated formatting or wording churn out of the same pull request.
